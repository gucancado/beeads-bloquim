criar funcionalidade de anexos em tarefas

Objetivo:
Implementar a funcionalidade de anexar arquivos nas tarefas do sistema, com uma estrutura preparada para expansão futura, já que no futuro os arquivos poderão ser vinculados também a outros tipos de entidades além de tarefas.

Requisitos funcionais:

1. Tipos de arquivo

* Deve ser possível anexar qualquer tipo de arquivo.
* Exemplos: imagens, PDFs, documentos, planilhas, vídeos, áudios, arquivos compactados e outros.

2. Formas de anexar arquivos
   Os arquivos poderão ser anexados pelos seguintes métodos:

* Botão "Anexar arquivo"

  * Deve existir um botão visível para selecionar arquivo manualmente.

* Drag and drop

  * O usuário pode selecionar um arquivo no computador e arrastar para:

    * a tela de edição de tarefa
    * a tela de edição do nó
  * Ao arrastar, a interface deve indicar visualmente que a área aceita arquivos.

* Ctrl+V / colar

  * Deve ser possível colar arquivos copiados previamente do computador.
  * Deve funcionar também com imagens copiadas para a área de transferência.
  * Exemplo: o usuário tira um print da tela e pressiona Ctrl+V para anexar essa imagem à tarefa.

3. Exibição dos anexos

* Os anexos devem ser exibidos abaixo da descrição da tarefa.
* Deve ser possível:

  * adicionar novos anexos
  * remover anexos existentes
* A estética visual dessa seção deve seguir uma linguagem semelhante à usada atualmente na área de aprovações dentro da tela de edição de tarefa.

4. Comportamento esperado

* O anexo deve ficar vinculado à tarefa atual.
* Ao salvar ou atualizar a tarefa, os anexos devem permanecer salvos e visíveis.
* A remoção de um anexo deve atualizar a interface imediatamente.
* A interface deve suportar múltiplos anexos na mesma tarefa.

Requisito de arquitetura importante:
A modelagem não deve ser criada de forma acoplada exclusivamente à tarefa.

Precisamos de uma estrutura escalável para que, no futuro, um arquivo possa ser vinculado também a outras entidades, como por exemplo:

* espaço de trabalho
* plano
* comentário de tarefa
* aprovação
* conteúdo inline dentro da descrição da tarefa
* outros tipos futuros

Por isso, implemente a estrutura de anexos de forma genérica e reutilizável, com uma relação de vínculo flexível entre arquivo e entidade.

Sugestão de modelagem:

* Criar uma entidade de arquivo/anexo separada
* Criar uma estrutura de vínculo que permita associar o arquivo a diferentes tipos de entidade
* Exemplo conceitual:

  * arquivo
  * tipo de vínculo
  * id da entidade vinculada

Não precisa implementar agora os outros vínculos além de tarefa, mas a arquitetura já deve nascer preparada para isso.

Requisitos de UX

* O usuário deve entender com clareza onde anexar arquivos.
* Ao arrastar arquivo sobre a tela, exibir feedback visual de dropzone.
* Ao colar arquivo com Ctrl+V, o sistema deve reconhecer automaticamente e criar o anexo.
* Os anexos devem ter uma apresentação organizada, limpa e consistente com o restante do sistema.
* Cada anexo deve ter ação de remoção.
* Se fizer sentido, mostrar nome do arquivo, tipo e tamanho.

Regras e cuidados

* Não limitar a funcionalidade apenas a imagens.
* O suporte a colar com Ctrl+V deve funcionar tanto para arquivos quanto para imagens vindas da área de transferência.
* A funcionalidade deve funcionar tanto na edição da tarefa quanto na edição do nó, desde que o anexo fique vinculado à tarefa correspondente.
* Estruturar backend, banco e frontend de forma reutilizável.

Critérios de aceite

* Consigo anexar um arquivo clicando em "Anexar arquivo".
* Consigo anexar um arquivo arrastando para a tela de edição de tarefa.
* Consigo anexar um arquivo arrastando para a tela de edição do nó.
* Consigo anexar uma imagem com Ctrl+V após tirar um print.
* Os anexos aparecem abaixo da descrição da tarefa.
* Consigo remover um anexo.
* Consigo ter múltiplos anexos na mesma tarefa.
* A estrutura criada permite futuramente associar arquivos a outras entidades sem refazer a base da funcionalidade.
